# システム設計書 全章統合概要

---

## 1. プロジェクト概要・目的

本システムは、
**血統情報（5 代血統）と競走成績データを統合的に学習する AI モデル**を構築し、

- レース単位での **勝率・複勝率の確率予測**
- 血統に基づく **競走能力・多次元適性の定量評価**

を同時に実現することを目的とする。

従来の経験則・主観評価に依存してきた競馬分析を、
**再現性・客観性・拡張性を備えたデータ駆動型意思決定基盤**へ昇華させることが主眼である。

---

## 2. 課題設定と設計思想

- 血統 × 成績 × レース条件の関係は **非線形かつ高次元**
- 明示的ルールでは捉えきれない相互作用（ニックス、成長曲線等）が存在
- 単一タスク最適化では、競馬の複雑性を十分に表現できない

これらを踏まえ、

- **Transformer 系モデル**
- **マルチタスク学習**
- **Embedding による高次元カテゴリ処理**

を中核とした設計思想を採用。

---

## 3. システム全体構成

- 入力

  - 血統情報（5 代血統・系統クロス・ニックス）
  - 競走成績情報（レース条件・馬体・走破結果）

- 中核モデル

  - TabTransformer / Transformer Encoder
  - 血統ブロック + 成績ブロックの統合

- 出力

  - レース予測（確率）
  - 血統由来の能力・適性スコア

**学習・推論・再学習を前提とした MLOps 対応構成**を採る。

---

## 4. 入力特徴量設計（Input Features）

### 血統特徴量

- 5 代血統表（31 ノード）を完全数値化
- 個体 ID・系統 Embedding
- 血量割合
- 父系・母系・父母間クロス one-hot
- ニックス Embedding（父 × 母系）

血統を「木構造」ではなく、**意味を保った固定長ベクトル**として扱う点が特徴。

### 競走成績特徴量

- レース条件（距離、路盤、馬場、季節、格）
- 出走馬属性（年齢、性別、脚質、馬体重）
- 結果情報（着順、正規化着順、タイム、上がり）

**カテゴリ＋連続値の混在を前提とした設計**。

---

## 5. 出力特徴量設計（Target Features）

### 5.1 レース予測タスク

- 出走馬ごとに

  - 勝率
  - 複勝率

- 出走頭数 N に対し N×2 の確率行列

**確率校正・期待値計算への応用を前提**。

### 5.2 能力・適性推定タスク

- 血統由来の能力・適性を **50 次元の連続スコア**として出力
- 距離、路盤、馬場、競馬場、脚質、成長、季節等を網羅
- 各スコアは独立で、合計 1 制約なし

**血統を「潜在能力空間」に写像する役割**を担う。

---

## 6. 学習データ・教師信号設計

- レース結果から

  - 勝敗ラベル
  - 複勝ラベル
  - 正規化順位

- 能力・適性は

  - 成績分布・条件別パフォーマンスから間接的に学習

**直接ラベルが存在しない概念を、マルチタスクで内包的に獲得**する設計。

---

## 7. 前処理・特徴量生成パイプライン

- 欠損・取消・外れ値の厳密除外
- カテゴリ変数は Embedding 前提でラベル化
- 連続値は Z-score 正規化
- 時系列成績のみ Position Encoding を適用

**学習・推論の完全再現性を担保**。

---

## 8. モデル設計

- Transformer Encoder を中核
- 血統ブロックと成績ブロックを統合
- マルチヘッド出力
- 動的タスク損失重み調整

非線形相互作用・長距離依存を同時に学習可能な構造。

以下は、提示内容から**冗長・説明的な日本語を排除**し、**である調・設計書用途**として整理した文章である。

# データベース設計

本設計は、競馬データの収集・正規化・血統特徴量生成・機械学習利用を前提としたデータベース構造を定義するものである。
データベースは用途ごとに分割し、責務を明確にする。

---

## race.db

### テーブル : races

レース単位の静的メタ情報を保持する。

| カラム名        | 型      | 説明               |
| --------------- | ------- | ------------------ |
| race_key        | TEXT    | レース一意識別キー |
| date            | TEXT    | 開催日             |
| venue_name      | TEXT    | 開催競馬場名       |
| race_number     | INTEGER | レース番号         |
| race_name       | TEXT    | レース名           |
| track_type      | TEXT    | 馬場種別           |
| distance        | INTEGER | 距離（m）          |
| grade           | TEXT    | 格付               |
| weather         | TEXT    | 天候               |
| track_condition | TEXT    | 馬場状態           |

---

### テーブル : horse_entries

レースと出走馬の関係および結果・属性情報を保持する事実テーブルである。

| カラム名       | 型      | 説明                   |
| -------------- | ------- | ---------------------- |
| race_key       | TEXT    | races テーブル参照キー |
| horse_id       | INTEGER | 馬 ID                  |
| finish_order   | INTEGER | 着順                   |
| frame          | INTEGER | 枠番                   |
| number         | INTEGER | 馬番                   |
| name           | TEXT    | 馬名                   |
| sex            | TEXT    | 性別                   |
| age            | INTEGER | 年齢                   |
| jockey_name    | TEXT    | 騎手名                 |
| weight_carried | REAL    | 斤量                   |
| time           | TEXT    | 走破タイム             |
| margin         | TEXT    | 着差                   |
| passing_order  | TEXT    | 通過順                 |
| last_3f        | REAL    | 上がり 3F              |
| speed_index    | REAL    | スピード指数           |
| popularity     | INTEGER | 人気順                 |
| body_weight    | INTEGER | 馬体重                 |
| diff_from_prev | INTEGER | 前走比増減             |
| trainer_name   | TEXT    | 調教師名               |
| trainer_region | TEXT    | 所属                   |
| owner          | TEXT    | 馬主                   |
| breeder        | TEXT    | 生産者                 |

---

## pedigree.db

### テーブル : pedigree_info

競走馬単位で血統構造および父系系統情報を保持する。

#### 基本情報

| カラム名           | 型   | 説明           |
| ------------------ | ---- | -------------- |
| horse_id           | TEXT | 馬 ID          |
| name               | TEXT | 馬名           |
| five_gen_ancestors | TEXT | 5 代祖先名一覧 |

#### 父系 31 ポジション情報

父系インデックス定義に基づき、31 ポジション分の種牡馬名および系統名を保持する。

| カラム名        | 型   | 説明                       |
| --------------- | ---- | -------------------------- |
| sire*name*\*    | TEXT | 対応ポジションの種牡馬名   |
| lineage*name*\* | TEXT | 対応ポジションの父系系統名 |

※ 上記セットが定義済み 31 ポジション分存在する。

---

## sire_lineage.db

### テーブル : sire_lineage

種牡馬単位の父系系統判定結果をキャッシュとして保持する。

| カラム名   | 型   | 説明       |
| ---------- | ---- | ---------- |
| horse_id   | TEXT | 種牡馬 ID  |
| horse_name | TEXT | 種牡馬名   |
| lineage    | TEXT | 父系系統名 |
| lineage_id | TEXT | 系統識別子 |

---

## 設計方針

- race.db はレース結果という事実データを正規化して保持する
- pedigree.db は血統構造を特徴量生成に適した形で固定化する
- sire_lineage.db は父系系統判定結果を再利用可能な知識キャッシュとして扱う
