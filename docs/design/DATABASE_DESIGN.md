# 競馬血統AIシステム データベース設計書（実データ反映版）

## 1. 全体構成

データは役割ごとに3つのSQLiteファイルに分割され、`race_key` または `horse_id` を介して論理的に結合されます。

- **ファイルパス**: `data/raw/` 以下の各ディレクトリ

| DBファイル名      | 格納場所             | 主な内容                                |
| ----------------- | -------------------- | --------------------------------------- |
| `race.db`         | `data/raw/race/`     | レース条件、出走馬成績、タイム、着順    |
| `pedigree.db`     | `data/raw/pedigree/` | 5代血統名リスト、各世代の種牡馬・系統名 |
| `sire_lineage.db` | `data/raw/pedigree/` | 種牡馬ごとの系統名と系統ID（マスタ）    |

---

| `weather`         | TEXT | `晴`                 | 天候                      |

## 3. pedigree.db

競走馬の血統構成を管理します。

### 3.1 pedigree_info テーブル（5代血統詳細）

このテーブルは、特徴量抽出を高速化するために**5代前の祖先までフラットに展開**されています。

| カラム名             | 型   | サンプルデータ（クロワデュノール）   | 説明                  |
| -------------------- | ---- | ------------------------------------ | --------------------- |
| `horse_id`           | TEXT | `0001352760`                         | 馬識別ID（0埋め10桁） |
| `name`               | TEXT | `クロワデュノール`                   | 馬名                  |
| `five_gen_ancestors` | TEXT | `キタサンブラック,ブラックタイド...` | 62頭の祖先名CSV       |
| `sire_name_父`       | TEXT | `キタサンブラック`                   | 1代前（父）の馬名     |
| `lineage_name_父`    | TEXT | `Halo 系`                            | 1代前（父）の系統名   |
| `sire_name_父父`     | TEXT | `ブラックタイド`                     | 2代前（父父）の馬名   |
| `lineage_name_父父`  | TEXT | `Halo 系`                            | 2代前（父父）の系統名 |
| ...                  | ...  | ...                                  | 母母母母父まで展開    |

**※ 注意**: カラム名に `sire_name_母 父父父父` のように空白が含まれている箇所があるため、SQLクエリ時は `[カラム名]` 形式での指定が必要です。

## 4. sire_lineage.db

種牡馬名から、AIモデルが扱いやすい「系統ID」への変換マスタです。

### 4.1 sire_lineage テーブル（系統マスタ）

| カラム名     | 型   | サンプルデータ     | 説明                              |
| ------------ | ---- | ------------------ | --------------------------------- |
| `horse_id`   | TEXT | `0001155349`       | 種牡馬のID                        |
| `horse_name` | TEXT | `キタサンブラック` | 種牡馬名                          |
| `lineage`    | TEXT | `Halo 系`          | 系統名                            |
| `lineage_id` | TEXT | `RC-HA`            | **系統分類コード**（Embedding用） |

---

## 5. データ結合リレーションシップ

AIモデル（TabTransformer等）への学習データセット作成時は、以下のフローでデータを統合します。

1. **レース情報の特定**: `races` からレース条件を取得。
2. **出走馬の結合**: `race_key` を用いて `horse_entries` を結合（1レースに複数頭）。
3. **血統データの結合**: `horse_id`（INT ⇔ TEXT変換に注意）を用いて `pedigree_info` を結合。
4. **系統カテゴリの特定**: `pedigree_info` 内の各祖先名、または `sire_lineage` のマスタを利用して `lineage_id` を紐付け。

---

## 6. 実データに基づく特記事項

- **ID形式の不一致**: `race.db` では `horse_id` が `INTEGER` (1352760) ですが、`pedigree.db` では `TEXT` ('0001352760') として格納されています。結合時は `CAST` または文字列整形が必要です。
- **空白の混入**: `races` の `weather`（'晴 '）や `pedigree_info` の一部カラム名に空白が含まれています。データクレンジング処理が必要です。
- **欠損値**: `horse_entries` の `margin` における `'---'` など、非数値データの数値化（0.0への置換等）が特徴量エンジニアリングで必要となります。
